= 용어 및 정의 :hardbreaks: :nofooter: :icons: font :linkattrs: :imagesdir: ./media/

[role="lead"]
청구 기본 설정은 NetApp Console 라이선스 및 구독에 대한 결제, 송장 및 갱신 동작을 처리하는 방식을 제어합니다.  다음 두 가지 수준 중 하나에서 기본 설정을 지정합니다.

* *글로벌* — 직접 라이선스, 마켓플레이스 구독, Keystone 구독에 적용됩니다.
* *구독당* — 특정 마켓플레이스 또는 직접 라이선스 계약에만 적용됩니다.


청구 출처는 다음과 같습니다.

* *마켓플레이스 구독* - AWS Marketplace, Azure Marketplace 또는 Google Cloud Marketplace를 통해 청구됩니다.  클라우드 마켓플레이스에서 제공하는 청구 계정, 지역 및 갱신 옵션을 선택할 수 있습니다.
* *직접 라이선스* - NetApp 통해 청구가 이루어집니다.  NetApp Console 에서 라이선스 키, 송장 연락처 및 자동 갱신을 관리합니다.
* * Keystone 구독* — 청구는 Keystone 계약을 통해 이루어집니다.  NetApp Console 에서 송장 수신자와 비용 센터 태그를 관리합니다.


== 권한 청구 기본 설정을 변경하려면 *계정 관리자* 또는 *청구 관리자* 역할이 필요합니다.  운영자는 기본 설정을 볼 수 있지만 수정할 수는 없습니다.

== 청구 기본 설정

청구 환경 설정을 통해 다음을 수행할 수 있습니다. - 모든 서비스에서 단일 청구서 연락처 목록을 적용합니다. - 사용량을 비용 센터에 맞춥니다. - 자동 갱신 및 만료 알림을 관리합니다. - 새로운 Cloud Volumes ONTAP (CVO) 시스템이 올바른 청구 계정을 상속받도록 합니다.

= 청구 필드 :page-role: 참조

== 마켓플레이스 필드 * *마켓플레이스 공급자* — AWS, Azure 또는 Google Cloud.  * *청구 계정/구독 ID* — 요금을 청구할 클라우드 청구 계정입니다.  * *계약 기간* — 계약 기간 및 갱신 행동.

= 청구 옵션 및 정의 :page-role: 참조

== 약관 및 옵션 * ** NetApp 라이선스 우선** — 활성 NetApp 라이선스에 대한 요금이 청구됩니다.  추가 사용권은 마켓플레이스로 이전됩니다.  * **마켓플레이스 구독에만 해당** — 모든 사용료는 마켓플레이스 계약을 통해 청구됩니다.  * **PAYGO(Pay-As-You-Go)** — 실제 사용량에 따라 요금이 청구되는 유연한 구독입니다.  * **연간 계약** — 기간제 마켓플레이스 구독.  * **자동 갱신** — 계약 만료 시 자동으로 계약을 갱신합니다.  * **사용자 정의 CVO 구성** — 하이퍼스케일러당 여러 구독을 매핑하기 위한 고급 모드입니다.  * **직접 라이선스** — 마켓플레이스 용량 이전에 사용된 NetApp 관리 라이선스입니다.  * **하이퍼스케일러** — AWS, Azure 또는 Google Cloud 공급자.  * **변경 사항 확인 대화 상자** — 실수로 청구 내용이 변경되는 것을 방지하기 위해 업데이트를 적용하기 전에 나타납니다.
